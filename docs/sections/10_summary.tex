\begin{description}
    \item[\HOne] Para la hipótesis \HOne~se han usado los hiperparámetros de \emph{player} (anexo \ref{anexo:hyperPlayer}), \emph{pokemon} (anexo \ref{anexo:hyperPokemon}) y \emph{move} (anexo \ref{anexo:hyperMove}). Estos escogen las clases que se usan para representar el estado de la batalla. Para el campo de batalla se usa \emph{player}, para cada uno de los 12 Pokémon se usa \emph{pokemon} y para cada uno de los 4 movimientos \emph{move} (48 movimientos en total). Estos determinan el número de parámetros de la primera capa de la red neuronal que está en el rango de 60-82876.

    \item[\HTwo] Para la hipótesis \HTwo~se ha usado el hiperparámetro \emph{useSelfPlay}, que determina si el agente se entrena contra sí mismo o no. Se van guardando una ventana de los últimos 20 epochs y se juegan \emph{nEpisodes} partidas en total (se elige aleatoriamente el oponente de cada partida). También están relacionados con esta hipótesis los hiperparámetros \emph{useRandom} y \emph{useMaxDamage}, que determinan si el agente se entrena contra un oponente aleatorio o un oponente que siempre elige el movimiento que hace más daño. Es necesario que al menos uno de estos tres hiperparámetros esté activo para que el agente tenga un oponente contra el que entrenar (figura \ref{fig:training}).

    \item[\HThree] Para la hipótesis \HThree~ se ha usado el hiperparámetro \emph{nTeams}, que determina el número de equipos que se usan para entrenar. Se han probado 5 valores distintos: 1, 2, 10, 100 y inf. Para los valores finitos, se tiene una carpeta con los equipos creados y los primeros n para el jugador y su oponente que se usan en un formato cuya diferencia es que se elige el equipo. Para el valor infinito, se usa el formato \emph{gen9randombattle}, en el que el equipo se genera aleatoriamente para cada partida.
\end{description}

También se han usado otros hiperparámetros (tabla \ref{tab:hyperparameters}) para poder probar la variabilidad de los resultados en relación con los hiperparámetros de las hipótesis. Entre estos se incluye el tamaño de la red (\emph{actor} y \emph{critic}), si usar critic (\emph{critic} vacío), el número de episodios por epoch (\emph{nEpisodes}), el factor de descuento (\emph{gamma}) y la función de recompensa (\emph{reward}).

Como no hay tiempo disponible para probar todas las combinaciones (3520000 totales), se han probado 151 combinaciones de hiperparámetros. Cada una se entrena con una única semilla y el entrenamiento dura 1000 epochs. Una vez que acabado el entrenamiento se guardan las redes para poder jugar partidas para validación.

Podemos dividir las métricas obtenidas en dos tipos: las obtenidas durante el entrenamiento y las obtenidas después del entrenamiento. Durante el entrenamiento se obtienen las siguientes métricas de cada epoch:

\begin{itemize}
    \item El porcentaje de victoria del agente de todos los episodios jugados en ese epoch. Puede no indicar que un agente sea más fuerte si este se entrena contra oponentes más débiles.
    \item La pérdida media del actor por todos los turnos en ese epoch.
    \item La recompensa media del agente por todos los turnos en ese epoch.
    \item El número medio de turnos por todos los episodios jugados en ese epoch.
    \item Si se usa crítico, la pérdida media del crítico por todos los turnos en ese epoch.
    \item Si se usa crítico, la recompensa media del crítico por todos los turnos en ese epoch.
\end{itemize}

Una vez que se tienen modelos, se pueden jugar partidas entre ellos para obtener una matriz de victorias. La imagen generada por la matriz es tan grande que no se puede incluir en el documento, pero se puede consultar en el pie de página\footnote{Porcentaje victorias: \url{https://drive.google.com/file/d/1FIfLRBHuJ33bC_FBJho3Hk0I-aJ2glsp/view?usp=sharing}}. La fila representa el porcentaje de victorias contra el modelo de la columna. Al seguir el mismo orden de filas y de columnas, dos casillas opuestas suman 1. La diagonal se rellena para comprobar que se han jugado suficientes partidas y so porcentaje de victorias es cercano a 0.5.

Esta matriz tiene todos los experimentos más el jugador aleatorio y el jugador que siempre elige el movimiento que hace más daño como referencias. Inicialmente cada celda se rellena con 10 partidas, pero posteriormente se van jugando más partidas para las casillas que tienen más incertidumbre.

La incertidumbre se estima mediante el intervalo de Wilson con un 95\% de confianza, utilizando la amplitud del intervalo (diferencia entre el límite superior y el inferior). En total se han jugado $3711670$ partidas. Los valores de incertidumbre también se puede consultar en el pie de página\footnote{Incertidumbre: \url{https://drive.google.com/file/d/1eJBiV4Dtwemnvf0MvkRXA4L6FiXguRy6/view?usp=sharing}}.

Con la matriz de victorias, ya se puede intentar aislar los efectos de cambiar un solo hiperparámetro. Para ello se hace una matriz de victorias para cada hiperparámetro, en la que cada celda representa la media del porcentaje de victorias del hiperparámetro de la fila contra el hiperparámetro de la columna. La diagonal está vacía y las casillas opuestas suman 0, porque si son negativas, es un porcentaje de derrotas. Un ejemplo de estas matrices es la figura \ref{fig:battlesMove}.

Una vez que se tiene la matriz de victorias, se puede aplicar un modelo Bradley-Terry~\cite{bradleyterry1952} para calcular una puntuación de fuerza para cada modelo. Esto es implementado manualmente usando un modelo de regresión logística. Con esto podemos ver que el modelo que siempre elige el movimiento de máxima potencia está notablemente por encima de los demás y que los experimentos pueden ser superiores al jugador aleatorio o ser inferiores a este. La tabla completa está en su propio anexo (Anexo~\ref{anexo:bradleyterry}).

Con las puntuaciones de Bradley-Terry, se puede intentar predecir la puntuación de fuerza de un modelo a partir de sus hiperparámetros. Para ello se ha entrenado un modelo \emph{GradientBoostingRegressor} de la librería \emph{scikit-learn} que será llamado modelo sustituto. Con las puntuaciones de nuevas combinaciones, se pueden probar nuevos modelos que predice que son fuertes.

También se puede intentar aislar un hiperparámetro y ver las puntuaciones relativas entre sus distintos valores de ese hiperparámetro. Se pueden generar matrices cuadradas iguales a las de victorias, pero al usar un modelo intermediario con solo $151$ datos de entrenamiento, no tiene porqué ser correcta. El modelo sustituto da una matriz completamente opuesta a la de victorias en el caso del hiperparámetro \emph{nTeams}.
