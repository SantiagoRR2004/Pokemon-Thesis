\subsection{\lang{Hypothesis}{Hipótesis} \HOne}

\ifthenelse{\equal{\LANG}{es}}{

}{}
\ifthenelse{\equal{\LANG}{en}}{
    Since the state representation depends on three hyperparameters, \emph{player}, \emph{pokemon}, and \emph{move}, the effect of each of them will be analyzed separately. However, appendix \ref{anexo:hyperNInputs} explains the distribution of the pseudo-hyperparameter \emph{nInputs}, which combines the three previous ones.
}{}

\ifthenelse{\equal{\LANG}{es}}{
    Si se considera el número de parámetros de entrada (\textit{nInputs}) y se representa la puntuación media de fuerza obtenida mediante el modelo de Bradley-Terry para cada representación, se obtiene la gráfica \ref{fig:BattlesNInputs}. Aún así, se analizarán los tres hiperparámetros por separado para ver si se pueden extraer conclusiones más concretas sobre cada uno de ellos.
}{}
\ifthenelse{\equal{\LANG}{en}}{

From it, the hypothesis that the number of input parameters alone explains the model's performance can be rejected, since no clear pattern is observed between the two variables. Even so, the three hyperparameters will be analyzed separately to see whether more specific conclusions can be drawn about each of them.
}{}

\ifthenelse{\equal{\LANG}{en}}{

The class with the higher number encompasses the class with the lower number, that is, it has all the attributes of the lower class plus some additional ones. \emph{S} means it is the most complex class and tries to represent everything without any limit on the number of parameters. For details of each representation, see appendices \ref{anexo:hyperPlayer}, \ref{anexo:hyperPokemon}, and \ref{anexo:hyperMove}.

\ifthenelse{\equal{\LANG}{es}}{

If the results of identical experiments that differ exclusively in this hyperparameter are compared (Figure \ref{fig:battlesPlayer}), it can be clearly seen that the more complex representation obtains a positive win rate against the simple representation. These results provide evidence in favor of the hypothesis proposed for this hyperparameter, so there are no reasons to reject it.

Overall, these results suggest that there is no clear relationship between representation complexity and the performance achieved by the model.
